Azure VM persistence issues when shut down is triggered.


6 Feb 2022

WARNING: My posts are in no way authoritative, I would even say some of it is speculative.
Azure VM disk's offer some disk caching options, namely:
  • Read/Write
  • Read-only
  • None
Detailed description can be found here .
Some context:
I was using Azure's VM (Linux) with Read/write mode, big mistake. I used it as a work machine and connected vis SSH over VSCode. I wrote some code, saved it, but I didn't push to GitHub, I know it was a mistake, usually I do not do this. From the desciption of Read/Write mode, it is the responsibility of the application to flush the cache. So, VSCode did not call fsync correctly as a consequence I lost my data from the disk when I shut down the VM.
This is a note to other people to make sure your application handles syncing of disk and memory if you decide to use Read/Write mode.
